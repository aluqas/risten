# risten Documentation

`risten` のドキュメントへようこそ。
本プロジェクトでは、議論・決定・仕様を以下の構造で管理しています。

## ディレクトリ構成

### 📂 `rfcs/` (Request for Comments)

**「未来」の提案と議論の場**
新しい機能や大規模な変更を提案する際は、ここに RFC を作成します。

- `0000-template.md`: テンプレート（未作成）
- 実装前の設計、他案との比較、メリット・デメリットを網羅的に議論します。

### 📂 `adr/` (Architecture Decision Records)

**「過去」の決定事項の記録**
設計上の重要な決定（「なぜその技術を選んだのか」）を記録します。

- 基本的に**不変（Immutable）**な記録です。
- 後から「なぜこうなっているのか」を振り返るために使用します。

### 📂 `research/` (Research & Context)

**「背景」となる調査資料**
RFC や ADR の根拠となる調査データ、他フレームワークとの比較などを置きます。

### 📂 `design/` (Current Design)

**「現在」の仕様書**
現在実装されているアーキテクチャや API の仕様を記述します。
実装が進むにつれて更新される、Living Document です。

### 📂 `archive/` (Archive)

**過去のメモ・思考ログ**
古くなった設計メモやブレインストーミングの記録です。歴史的経緯を知るために残されています。

---

## ワークフロー

1. **アイデア出し**: `archive/` や `research/` で調査・思考整理。
2. **提案 (RFC)**: `rfcs/` に新しいプロポーザルを作成。チームで議論し、設計を固める。
3. **決定 (ADR)**: 採用された方針の理由を `adr/` に記録（必要に応じて）。
4. **計画 (Roadmap)**: `ROADMAP.md` に実装タスクを追加。
5. **仕様化 (Design)**: 実装完了後、`design/` に現在の仕様として反映。
